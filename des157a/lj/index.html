Best Practices for Modals / Overlays / Dialog Windows

The article “Best Practices for Modals / Overlays / Dialog Windows ” by Naema Baskanderi details the recommended usage for assets that can either be a great solution or an irritating problem to the user. Modal windows are elements that remain on top of the main window.

Using modal windows can be beneficial when you need to alert the user about something important, to get their input, and display more information either in or out of context. Proper modal windows include some way of closing the modal, a useful title, clear buttons, an appropriate size and focus, and are the result of the user interaction.

Beware of using modal windows in mobile devices, as the size constraint of the device hinders the ability for modal elements to be used optimally. In addition, it is important to note accessibility of modal windows such as adding keyboard controls.

Best Practices for Form Design

In reading the article “Best practices for form design” by Salim Ansari, I learned many valuable concepts that I hadn’t thought of previously, and others that I could relate to as a frustrated user. As the title suggests, the article discusses best practices to increase the usability of a form. I began to think about the everyday forms that I fill out online: setting up an account, ordering items online, or applying for jobs.

One best practice I found particularly striking was regarding asking for sensitive information. If a phone number is an optional field, I will not input it. However, if there is an explanation for it, such as calling about a package delivery, I am more likely to enter my phone number.

A website form that I think exemplifies best practices is checking out via the Target mobile app. There are very few fields, and all of the action buttons are placed conveniently without the extra need to leave the form. There is also good use of color and font sizes to distinguish information from each other.
